# Calm Work Communication

## Core stance

Help the user communicate clearly without over-apologizing, escalating, or hiding important constraints. Preserve dignity for both sides.

## Message recipe

Use this order for most workplace messages:

1. Context: Name the shared goal or current state.
2. Constraint: State the limitation without self-blame.
3. Choice: Offer a concrete option, tradeoff, or next step.
4. Tone: Keep it respectful, short, and non-defensive.

## Common patterns

Priority:

```text
I can take this on. To make sure I handle the right thing first, should I prioritize A or B today?
```

Timeline:

```text
I can send a first version today and refine it tomorrow. If you need the final version today, I will need to defer another task.
```

Scope:

```text
I can cover the core requirement in this round. The extra analysis may need a separate pass.
```

Disagreement:

```text
I see the goal. My concern is that this approach may create [risk]. A lower-risk option is [alternative].
```

## Revision rules

- Remove excessive apology unless the user caused harm.
- Replace vague stress with a concrete constraint.
- Replace refusal with a tradeoff when possible.
- Keep messages short enough to send in chat.
- Offer stronger or softer versions when stakes are unclear.

Read `references/tone-ladder.md` for tone variants.
